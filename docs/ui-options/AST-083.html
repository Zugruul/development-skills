Iterative UI mode is ON (the agent waits for your pick before committing to a UI, and keeps building everything else meanwhile). Turn it off with touch .claude/ITERATIVE_UI_OFF in the repo — or just tell the agent "turn off iterative UI" / that you'll be AFK, and it will decide UIs itself.
Ways back to the agent: comment on the issue (read every iteration, works even after the session ends), or reopen the building session in a terminal: claude --resume 0eac8db4-c31f-4f51-9512-bc7a5098189a.

Option A — Inline rail

Progress lives inside the chat bubble as a slim rail (state + stage + %). On completion the bubble collapses to an "opened in panel" chip and the artifact panel detaches beside it. Failure turns the same bubble amber with an inline trace link. Voice queue mirrors the rail.

Option B — Voice-panel queue

Chat stays clean — one acknowledgment line. All progress lives in the voice-panel queue indicator (§12.5) as an expandable job with a progress ring. Completion opens the panel + TTS announce; failure flips the queue chip amber and drops a trace-linked line into chat.

Option C — Mini-preview morph

The chat bubble hosts a mini-preview tile — a live thumbnail that scans/forms as the render runs, then morphs (shared-element) into the full detached artifact panel on completion. Failure turns the same tile into an error tile with a trace link. The affordance and the panel are one continuous object.
