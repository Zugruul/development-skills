## §8.5.2 generalization contract — AMENDED

- **§8.5.2 generalization contract** — THE SYSTEM SHALL enforce, at emission time, that project specifics never leave the feed: an item's `generalized` field, when non-empty, together with its `summary`, SHALL NOT contain the iteration's own task id or an issue/PR reference — bare (`#<digits>`) OR qualified (`<slug>#<digits>`, e.g. `comm-platform#71`) — such a record SHALL be rejected with an actionable error rather than appended. An item whose `generalized` field is empty SHALL be treated as local-only and SHALL be routable only as `ignore`.

## §8.5.6 qualified references — ADDED

- **§8.5.6.1 emit-time qualification** — WHEN `feedback.py emit` validates and appends a record THE SYSTEM SHALL normalize every bare `#<digits>` reference in that record's `items[].evidence[]` and `items[].routing.ref` to `<project.name>#<digits>`, where `project.name` is read from the EMITTING repo's own `.claude/project.yaml` (`config.py`'s loader) — never from the record's own content. A reference already qualified by ANY project (`<slug>#<digits>`, where `<slug>` is a run of word/hyphen characters immediately preceding the `#` with no intervening whitespace) SHALL pass through verbatim, regardless of which project's slug it carries.

- **§8.5.6.2 route-time qualification** — `feedback.py route`'s `ref` argument SHALL be normalized by the same rule as §8.5.6.1 before being written into `items[idx].routing.ref`.

- **§8.5.6.3 idempotence** — Re-running qualification (emit, route, or the migration of §8.5.6.4) over already-qualified text SHALL change nothing — a qualified reference SHALL NOT be re-qualified or double-prefixed.

- **§8.5.6.4 one-time migration** — THE SYSTEM SHALL provide `feedback.py migrate-qualify`, a one-shot command that qualifies every bare `#<digits>` already present in an EXISTING feed file's `items[].evidence[]` and `items[].routing.ref`, using the same rule as §8.5.6.1. The migration SHALL be applied as surgical, line-level text edits that leave every other byte of the feed file untouched (comments, quoting style, and all fields outside `evidence[]`/`routing.ref` — including `summary`, `generalized`, and `detail` — are never rewritten). The migration SHALL be idempotent per §8.5.6.3.

## §9 Invariants — AMENDED

- The feedback generalization ban (§8.5.2) rejects a qualified reference (`<slug>#N`) exactly as it rejects a bare one (`#N`) — neither form of project-specific reference may leave the feed via `summary`/`generalized`.
- Feedback reference qualification (§8.5.6) never rewrites a reference already qualified by any project, and never touches `summary`, `generalized`, or `detail` — only `evidence[]` and `routing.ref` are in scope.
