## §8.6 work.type / work.sync — PR-less local delivery + board-sync policy — ADDED

`work` (optional; absent == `{ type: pr }`) SHALL configure a task's delivery mode and, for local delivery, WHEN queued board mutations flush:

```yaml
work:
    type: pr | local
    sync:            # only valid with type: local; absent == { mode: realtime }
        mode: realtime | task-close | session-end | manual
```

**work.type** governs delivery. `pr` (default) — push the branch, `gh pr create`, PR review, `gh pr merge`; unchanged from today's behavior. `local` — the branch stays local: review is `git diff <mainBranch>..<branch>` instead of a PR diff; approval is recorded as an ISSUE comment (not a PR review); the orchestrator squash-merges locally with role attribution (the same Applied-by/Reviewed-by/Co-authored-by recipe as PR delivery) and pushes `<mainBranch>` — the push SHALL be skipped, with a stated reason, if the repo has no remote; the board announce carries the merge SHA rather than a PR link. `methodology.autoMerge: false` combined with `work.type: local` SHALL leave the branch unmerged at *In review* for a human, mirroring the human-approves-a-PR path without a PR to wait on.

**work.sync** (rejected — `validate-config.py` — when `work.type` is `pr`) governs WHEN queued board mutations flush, never WHETHER they happen. `scripts/work-mode.sh should-sync <event>` (event ∈ `transition`, `task-close`, `session-end`, `blocked`, `new-item`) resolves `now` or `defer` per this matrix:

| mode        | transition | task-close | session-end | blocked | new-item |
|-------------|-----------|------------|--------------|---------|----------|
| realtime    | now       | now        | now          | now     | now      |
| task-close  | defer     | now        | defer        | now     | now      |
| session-end | defer     | defer      | now          | now     | now      |
| manual      | defer     | defer      | defer        | now     | now      |

SAFETY VALVE: `blocked` and `new-item` events SHALL ALWAYS resolve `now`, in every mode — a human SHALL never wait through a batching window to learn a task is blocked or that a new item exists. The deferred-mutation queue/replay mechanism itself is `board-queue.sh` (#77, unmodified by this task); a caller that receives `defer` simply skips the `board.sh` call until the next matching sync point, where the batched calls run (self-queuing again if rate-limited).

`config.py get work.type` / `config.py get work.sync.mode` SHALL resolve to `pr` / `realtime` respectively whenever the corresponding key is absent from an existing config, so every bash caller sees a value without special-casing absence.

## §9 Invariants — ADDED

- `work.sync` is only accepted under `work.type: local`; `validate-config.py` rejects it under `type: pr` (including the implicit default) with an actionable message.
- A `blocked` or `new-item` board-sync event is never deferred, regardless of `work.sync.mode` — the safety valve is unconditional.
