## §8.5.1 emission — AMENDED

- **§8.5.1 emission** — WHEN a build-loop iteration ends AND `methodology.feedback` is enabled THE SYSTEM SHALL collect structured feedback about the WORKFLOW (never the project being built) as a `loop-feedback` record — `schemaVersion`, a canonical quoted ISO-8601 `ts` string (see §8.5.7), `iteration` (task, outcome, reviewRounds), `source` (role, model), and one or more `items[]` each carrying `category` (worked-well/friction/incident/recommendation), `area`, `severity`, `summary`, and a `generalized` restatement — and append it to the configured feed (`methodology.feedback.feed`, default `.claude/feedbacks/feed.yaml`).

## §8.5.7 ts identity and normalization — ADDED

- **§8.5.7.1 ts is the routing key** — A record's `ts` field is its identity for `feedback.py route`; two records sharing a `ts` make routing ambiguous, so `emit` SHALL reject a `ts` that already exists in the feed (compared per §8.5.7.2).

- **§8.5.7.2 emit-time normalization** — WHEN `feedback.py emit` loads a record whose `ts` was re-typed by the YAML loader into a `datetime`/`date` object (the result of an unquoted ISO-8601 timestamp in the source file) THE SYSTEM SHALL normalize it to the canonical string form `%Y-%m-%dT%H:%M:%SZ` (UTC assumed for a naive value, converted for an aware one) BEFORE the duplicate-`ts` check and BEFORE the record is dumped to the feed. A `ts` that is already a string passes through unchanged. The dumped feed line SHALL be quoted (e.g. `ts: '2026-07-01T12:00:00Z'`), matching every hand-authored record.

- **§8.5.7.3 route-time normalization** — `feedback.py route`'s `ts` lookup SHALL apply the same normalization (§8.5.7.2) to each feed record's `ts` before comparing it against the CLI `ts` argument, so a legacy record already in the feed with a datetime-typed (unquoted) `ts` remains addressable by its equivalent ISO-8601 string without a hand-edit of the archive.

## §9 Invariants — AMENDED

- A feed record's `ts` is always dumped and compared in its normalized, quoted ISO-8601 string form (§8.5.7) — an unquoted timestamp that YAML re-types to a `datetime`/`date` object is never left as such in the feed, and never causes a record to become unroutable by `route`.
